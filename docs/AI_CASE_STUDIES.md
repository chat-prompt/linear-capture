## Day 1-4: 4일 만에 Linear 자동화 macOS 앱 완성

*2026-01-15*

### 한줄 요약
스크린샷 캡처만 하면 AI가 자동으로 Linear 이슈를 생성해주는 macOS 앱을 Claude Code와 함께 4일 만에 만들었습니다. 하루 20-40분 절약하고, 이슈 품질도 올라갔습니다.

### 이런 분들께 도움돼요
- Linear 쓰는데 이슈 작성이 귀찮아서 자꾸 미루는 PM/기획자
- 팀의 업무 기록이 여기저기 흩어져서 관리가 안 되는 분
- AI 도구로 업무 자동화를 시도해보고 싶은데 어디서부터 시작할지 모르는 분

### 문제 상황 (Before)

우리 팀은 카톡, 채널톡, Slack 등 다양한 채널에서 업무가 발생했습니다. 문제는 이런 대화들이 Linear 이슈로 제대로 기록되지 않는다는 거였어요.

"아, 이거 이슈로 만들어야 하는데..." 하면서도 귀찮아서 미루다 보면:
- 실제로 수행한 업무와 Linear에 기록된 내용 간의 괴리가 계속 커졌고
- Linear가 팀의 실제 업무 맥락을 충분히 담지 못해 AI가 업무를 이해하고 돕기 어려웠어요
- 그 결과, 조직 차원의 업무 파악 / 개선 / 자동화가 불가능해졌습니다

특히 스크린샷 기반으로 이슈를 만들 때가 가장 번거로웠어요:
1. 스크린샷 찍기
2. Linear 열기
3. 이미지 업로드하기
4. 제목 직접 작성하기
5. 설명 직접 작성하기
6. 프로젝트, 담당자, 우선순위 등 선택하기

**한 번에 3-5분씩 걸렸고, 하루에 5-10번 정도 반복**했습니다.

### 사용한 도구
- **도구명**: Claude Code (CLI)
- **모델**: Claude Opus 4.5
- **AI 분석**: Google Gemini 2.5 Flash Lite, Anthropic Claude Haiku 4.5
- **특이사항**: Electron, TypeScript 몰라도 Claude가 다 설명해주고 코드 작성

### 작업 과정

#### Linear Capture 앱 기본 구조 만들기

첫날, Claude에게 이렇게 요청했습니다.

> macOS용 스크린샷 캡처 → Linear 이슈 자동 생성 앱을 만들고 싶어

저는 Electron이 뭔지도 몰랐지만, Claude가 전체 구조를 잡아줬어요. 단축키(⌘+Shift+L)를 누르면 화면 캡처가 시작되고, 이미지를 Cloudflare R2에 업로드한 뒤, 이슈 생성 폼이 뜨는 구조였습니다.

#### AI가 스크린샷을 분석해서 이슈 정보 자동 생성

기본 구조가 완성되자, 다음 단계로 넘어갔습니다.

> 스크린샷을 AI로 분석해서 이슈 제목, 설명, 프로젝트, 담당자까지 자동으로 채워주면 좋겠어

Gemini Vision API를 활용해서 스크린샷의 텍스트를 읽고, 내용을 분석해서 자동으로:
- **이슈 제목** 생성 (예: "로그인 - 소셜로그인 세션 만료 오류 수정")
- **마크다운 형식 설명** 작성 (## 이슈 / ## 상세 내용 / ## To Do)
- **관련 프로젝트** 자동 선택
- **담당자** 매칭
- **우선순위** 추천 (긴급/높음/보통/낮음)
- **작업 포인트** 추정 (1/2/3/5/8)

#### DMG 패키징의 늪... 그리고 탈출

앱이 잘 작동하니 이제 배포할 차례였습니다. 하지만 여기서 큰 문제가 생겼어요.

개발 모드(`npm run start`)에서는 모든 게 완벽하게 작동했는데, **DMG로 패키징해서 설치하면 앱이 아예 실행되지 않았습니다.**

Claude가 4가지 원인을 모두 찾아서 하나씩 해결해줬고:
1. **네이티브 모듈 문제**: Electron 내장 API로 대체
2. **파일 권한 문제**: `app.getPath('userData')` 경로 사용
3. **Hardened Runtime 충돌**: `hardenedRuntime: false` 설정
4. **Gatekeeper 차단**: Finder에서 우클릭 → "Open" 안내

### 결과 (After)

| 항목 | Before | After |
|------|--------|-------|
| 소요 시간 | 3-5분 (수동 작업) | 30초 (단축키 + AI 자동) |
| 하루 작업 횟수 | 5-10번 | 5-10번 |
| **하루 절약 시간** | - | **20-40분** |
| 이슈 품질 | 제목만 있거나 설명 부실 | 맥락 있는 설명 + To Do 자동 생성 |
| Linear 활용도 | Slack/카톡에만 공유하고 끝 | 모든 업무가 Linear 이슈로 기록 |

### AI 활용 팁

#### 효과적이었던 것
1. **실패해도 괜찮다, 단계별로 다시 시도하면 됨** - AI와 협업할 때 한 번에 다 하려고 하지 말고 **작게 쪼개서 하나씩 검증**하는 게 중요합니다
2. **문제 상황을 구체적으로 설명하면 해결책이 나옴**
3. **AI에게 충분한 컨텍스트를 주면 정확도가 올라감**
4. **개발자 아니어도 Claude Code로 앱 만들 수 있다**

#### 이렇게 하면 안 돼요
1. **한 번에 너무 많은 기능을 추가하지 마세요**
2. **에러 메시지를 Claude에게 안 보여주지 마세요**
3. **개발 모드에서만 테스트하지 마세요**

#### 커스텀 단축키
> 단축키도 바꿀 수 있으면 좋겠어

**팀원 C**: "업데이트했는데 예전 버전이 계속 실행돼요"

- 이전 버전 앱이 완전히 삭제 안 된 경우

- DMG 마운트 시 볼륨 이름 충돌 (Linear Capture 1, 2...)

```bash

```

## Slack/Notion 연동: Linear를 SSOT로

*2026-01-29*

### TL;DR

Linear 이슈만 보면 맥락을 알 수 없어서 매번 Slack, Notion을 뒤져야 했습니다. "이슈 하나에 관련 문서 다 붙이면 안 되나?"라는 생각으로 Slack/Notion 연동을 시작했고, OAuth deep link 패턴 하나 잡으니 두 서비스 모두 빠르게 연동할 수 있었습니다.

### 배경: 왜 이걸 만들었나

**문제**: Linear에서 이슈를 열면 "이게 뭐였더라?" 싶을 때가 많습니다.

- Slack에서 논의한 내용이 있었는데... 어디였지?
- Notion에 정리해둔 스펙 문서가 있었는데... 뭐였지?
- 나중엔 Gmail 스레드도 찾아야 하고...

**목표**: Linear가 진정한 SSOT(Single Source of Truth)가 되도록, 이슈 생성 시 관련 Slack 메시지, Notion 문서를 자동으로 검색해서 첨부하기.

### 작업 하이라이트

#### OAuth 연동 - deep link 패턴의 위력

Electron 앱에서 OAuth를 구현하는 건 생각보다 까다롭습니다.

**해결책: deep link (커스텀 프로토콜)**

```
linear-capture://oauth/slack/callback?code=xxx
```

1. 시스템 브라우저에서 Slack 로그인
2. 콜백 URL을 `linear-capture://` 스킴으로 설정
3. macOS가 해당 URL을 우리 앱으로 라우팅
4. 앱에서 code를 받아 토큰 교환

이 패턴을 Slack에서 한번 잡으니까, **Notion도 거의 복붙 수준으로 끝났습니다.**

#### Notion Search API의 함정

**문제**: Notion Search API는 **페이지 제목만** 검색합니다. 본문 내용은 검색이 안 돼요.

**현재 해결책**: Blocks API로 페이지 본문을 별도 추출하고 있습니다.

### 현재 상태

- Slack 검색: 동작함
- Notion 검색: 제목 검색은 됨, 본문 검색은 개선 필요
- Gmail: 아직 미연동

## i18n: 5개 언어 다국어화

*2026-02-02*

### 한줄 요약
Electron 앱을 AI와 함께 3일 만에 5개 언어로 다국어화했습니다. 프론트엔드뿐 아니라 백엔드 AI 프롬프트까지 번역해서 진짜 글로벌 앱을 만들었습니다.

### 문제 상황 (Before)

Linear Capture라는 macOS 앱을 만들었습니다. UI가 전부 한국어였습니다.

문제는 **Linear를 쓰는 사람들은 한국인보다 영어권 사용자가 훨씬 많다**는 것이었습니다. 글로벌 서비스인데 한국어 앱만 만들어놓으니 타겟 사용자의 대부분을 놓치고 있었습니다.

처음엔 "영어 하나만 추가하자"고 생각했는데, **어차피 다국어 처리하는 김에 5개 언어를 동시에 처리해도 AI 리소스가 크게 다르지 않을 것 같았습니다**. 한국어, 영어, 독일어, 프랑스어, 스페인어 - 5개 언어를 한 번에 처리하기로 했습니다.

### 사용한 도구
- **도구명**: OpenCode (시지프스 에이전트)
- **보조 에이전트**: Librarian (라이브러리 조사 전문)
- **특이사항**: Git Worktree로 Gmail 연동 기능과 병렬 작업

### 작업 과정

#### 라이브러리 조사부터 - Librarian 에이전트의 활약

> i18next + Electron 조합의 베스트 프랙티스 찾아줘

Librarian이 공식 문서, Stack Overflow, 실제 오픈소스 프로젝트를 다 뒤져서 정리해줬습니다.

#### 5개 언어 번역 파일 한 번에 생성

> 다국어 지원 구현해줘. 한국어랑 영어부터 시작하고, 독일어/프랑스어/스페인어도 추가해줘

5개 언어 × 150개 이상의 문자열이었습니다. 수동으로 했으면 며칠은 걸렸을 작업이 한 번에 끝났습니다.

#### 백엔드 프롬프트까지 다국어화 - 진짜 다국어 앱의 조건

**앱 UI만 번역하면 진짜 다국어 앱이 아닙니다.**

> AI 분석 결과도 선택된 언어로 나오게 해줘

이제 독일어 사용자가 앱을 쓰면 UI도 독일어, AI 분석 결과도 독일어로 나옵니다. **프론트엔드 + 백엔드 전체 스택 다국어화입니다.**

### Before vs After
| 항목 | Before | After |
|------|--------|-------|
| 지원 언어 | 한국어만 | 한국어, 영어, 독일어, 프랑스어, 스페인어 |
| AI 분석 결과 | 항상 한국어 | 사용자 언어에 맞춰 출력 |
| 타겟 사용자 | 한국 Linear 사용자 | 글로벌 Linear 사용자 |
| 작업 기간 | - | 3일 |

## Git Worktree: 병렬 개발

*2026-02-02*

### 한줄 요약

Git Worktree와 AI 코딩 도구를 조합해서 다국어 지원, Notion 연동, 검색 기능을 동시에 개발했고, 충돌 없이 깔끔하게 머지까지 완료했습니다.

### 문제 상황 (Before)

사이드 프로젝트로 Linear Capture라는 Electron 앱을 개발하고 있었는데요. 추가하고 싶은 기능이 한꺼번에 여러 개 생겼습니다.

- 다국어(i18n) 지원
- Gmail 연동
- Notion/Slack 검색 기능

급한 건 아니었지만, 하나씩 순차적으로 하면 시간이 오래 걸릴 것 같았어요.

### 작업 과정

#### Worktree가 뭔지부터 물어봤습니다

> 현재 작업하고 있는 linear-capture를 다양한 기능을 병렬 작업하고 싶은데 가능할까?

AI가 Worktree 개념을 쉽게 설명해줬어요. 핵심은 이거였습니다:

**"하나의 Git 저장소를 여러 폴더에서 동시에 작업하는 기능"**

#### 충돌 방지 전략까지 알아서 세워줬습니다

| 파일 | 상태 | 충돌 위험 |
|------|------|----------|
| `.sisyphus/` (계획 파일) | Git 추적됨 | ⚠️ 위험 |
| `.env` | gitignore | ✅ 안전 |
| `CLAUDE.md` | Git 추적됨 | ⚠️ 수정 시 충돌 가능 |

이 전략 덕분에 나중에 머지할 때 **충돌이 하나도 없었습니다**.

#### 터미널 3개로 동시 작업 시작

```
터미널 1: linear-capture/ [master]
터미널 2: linear-capture-worktrees/gmail-integration/ [feature/gmail-integration]
터미널 3: linear-capture-worktrees/i18n/ [feature/i18n]
```

각 터미널에서 OpenCode를 실행하면, 각각 독립적인 AI 세션이 돌아갑니다.
