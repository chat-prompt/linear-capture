# 비개발자가 Claude Code로 4일 만에 Linear 자동화 macOS 앱 만든 후기

## 📝 한줄 요약
스크린샷 캡처만 하면 AI가 자동으로 Linear 이슈를 생성해주는 macOS 앱을 Claude Code와 함께 4일 만에 만들었습니다. 하루 20-40분 절약하고, 이슈 품질도 올라갔습니다.

## 🎯 이런 분들께 도움돼요
- Linear 쓰는데 이슈 작성이 귀찮아서 자꾸 미루는 PM/기획자
- 팀의 업무 기록이 여기저기 흩어져서 관리가 안 되는 분
- AI 도구로 업무 자동화를 시도해보고 싶은데 어디서부터 시작할지 모르는 분

## 😫 문제 상황 (Before)

우리 팀은 카톡, 채널톡, Slack 등 다양한 채널에서 업무가 발생했습니다. 문제는 이런 대화들이 Linear 이슈로 제대로 기록되지 않는다는 거였어요.

"아, 이거 이슈로 만들어야 하는데..." 하면서도 귀찮아서 미루다 보면:
- 실제로 수행한 업무와 Linear에 기록된 내용 간의 괴리가 계속 커졌고
- Linear가 팀의 실제 업무 맥락을 충분히 담지 못해 AI가 업무를 이해하고 돕기 어려웠어요
- 그 결과, 조직 차원의 업무 파악 / 개선 / 자동화가 불가능해졌습니다

특히 스크린샷 기반으로 이슈를 만들 때가 가장 번거로웠어요:
1. 스크린샷 찍기
2. Linear 열기
3. 이미지 업로드하기
4. 제목 직접 작성하기
5. 설명 직접 작성하기
6. 프로젝트, 담당자, 우선순위 등 선택하기

**한 번에 3-5분씩 걸렸고, 하루에 5-10번 정도 반복**했습니다. 이대로는 Linear를 팀의 단일 원천(Source of Truth)으로 만들 수 없다고 판단했어요.

## 🛠️ 사용한 도구
- **도구명**: Claude Code (CLI)
- **모델**: Claude Opus 4.5
- **AI 분석**: Google Gemini 2.5 Flash Lite, Anthropic Claude Haiku 4.5
- **특이사항**: Electron, TypeScript 몰라도 Claude가 다 설명해주고 코드 작성

## 🔧 작업 과정

### Linear Capture 앱 기본 구조 만들기

첫날, Claude에게 이렇게 요청했습니다.

> macOS용 스크린샷 캡처 → Linear 이슈 자동 생성 앱을 만들고 싶어

저는 Electron이 뭔지도 몰랐지만, Claude가 전체 구조를 잡아줬어요. 단축키(⌘+Shift+L)를 누르면 화면 캡처가 시작되고, 이미지를 Cloudflare R2에 업로드한 뒤, 이슈 생성 폼이 뜨는 구조였습니다.

메인 프로세스, 렌더러 프로세스, IPC 통신... 용어는 어려웠지만 Claude가 하나씩 설명해주고 코드를 작성해줘서 금방 이해할 수 있었습니다.

---

### AI가 스크린샷을 분석해서 이슈 정보 자동 생성

기본 구조가 완성되자, 다음 단계로 넘어갔습니다.

> 스크린샷을 AI로 분석해서 이슈 제목, 설명, 프로젝트, 담당자까지 자동으로 채워주면 좋겠어

이게 정말 핵심 기능이었어요. Gemini Vision API를 활용해서 스크린샷의 텍스트를 읽고, 내용을 분석해서 자동으로:
- **이슈 제목** 생성 (예: "로그인 - 소셜로그인 세션 만료 오류 수정")
- **마크다운 형식 설명** 작성 (## 이슈 / ## 상세 내용 / ## To Do)
- **관련 프로젝트** 자동 선택
- **담당자** 매칭
- **우선순위** 추천 (긴급/높음/보통/낮음)
- **작업 포인트** 추정 (1/2/3/5/8)

처음엔 AI가 프로젝트를 잘못 매칭하는 경우가 있었는데, Claude에게 "프로젝트 설명도 AI에 컨텍스트로 제공하면 정확도가 올라갈 것 같아"라고 하니까 바로 개선해줬습니다. **충분한 컨텍스트를 주니 정확도가 확 올라갔어요.**

---

### DMG 패키징의 늪... 그리고 탈출

앱이 잘 작동하니 이제 배포할 차례였습니다. 하지만 여기서 큰 문제가 생겼어요.

개발 모드(`npm run start`)에서는 모든 게 완벽하게 작동했는데, **DMG로 패키징해서 설치하면 앱이 아예 실행되지 않았습니다.** 핫키를 눌러도 반응이 없었어요.

> 개발 모드에서는 핫키가 잘 작동하는데, DMG로 설치하면 앱이 실행이 안 돼

문제 상황을 구체적으로 설명하니, Claude가 체계적으로 분석을 시작했습니다:

1. **네이티브 모듈 문제**: C++ 바인딩을 사용하는 `mac-screen-capture-permissions` 모듈이 제대로 번들링되지 않았어요
   - 해결: Electron 내장 API(`systemPreferences`)로 대체

2. **파일 권한 문제**: crash 로그를 앱 번들 내부에 쓰려고 해서 권한 오류 발생
   - 해결: `app.getPath('userData')` 경로 사용

3. **Hardened Runtime 충돌**: Apple 인증서 없이 ad-hoc 서명할 때는 `hardenedRuntime: true`가 문제였어요
   - 해결: `hardenedRuntime: false` 설정

4. **Gatekeeper 차단**: macOS가 서명되지 않은 앱을 조용히 차단
   - 해결: Finder에서 우클릭 → "Open" 안내

Claude가 4가지 원인을 모두 찾아서 하나씩 해결해줬고, 드디어 DMG가 정상 작동했습니다!

---

### 실패하고 배운 것: Settings 기능 추가

앱이 작동하니 욕심이 생겼어요. 팀원마다 개인 Linear API 토큰을 설정할 수 있으면 좋겠다고 생각했습니다.

> 멤버별로 Linear API 토큰을 설정할 수 있게 Settings 기능을 만들어줘

Claude가 Settings UI, 토큰 저장소, IPC 핸들러까지 다 만들어줬어요. 개발 모드에서는 완벽했습니다.

하지만... **DMG로 빌드하니 앱이 또 실행되지 않았습니다.** 이번엔 아무런 오류 메시지도 없이 그냥 실행이 안 됐어요.

원인을 찾기 어려워서 일단 전부 롤백했습니다. 답답했지만, Claude가 이렇게 제안했어요:

> "Phase를 1-6으로 나눠서, 각 단계마다 DMG를 빌드하고 테스트하면서 신중하게 다시 구현해봅시다"

**이게 정말 중요한 교훈이었어요.** AI와 협업할 때 한 번에 다 하려고 하지 말고, **작게 쪼개서 하나씩 검증하는 게 훨씬 안전**하다는 걸 배웠습니다.

Phase 1: Settings 저장소 → DMG 테스트 ✅
Phase 2: Settings UI → DMG 테스트 ✅
Phase 3: IPC 핸들러 → DMG 테스트 ✅
Phase 4: Tray 메뉴 추가 → DMG 테스트 ✅
Phase 5: 메인 UI 버튼 추가 → DMG 테스트 ✅

각 단계마다 확인하니까 문제가 생겨도 바로 원인을 찾을 수 있었어요. Phase 4에서 트레이 아이콘이 안 보이는 문제가 있었는데, 바로 이전 단계로 돌아가서 고칠 수 있었습니다.

결국 Settings 기능을 안전하게 완성했고, 버전 1.1.0을 릴리스했습니다!

## 💬 이 과정에서 배운 AI 활용 팁

### 효과적이었던 것

1. **실패해도 괜찮다, 단계별로 다시 시도하면 됨**
   - Settings 기능 추가했다가 앱 망가졌을 때, Claude가 Phase 1-5로 나눠서 각 단계마다 DMG 테스트하며 안전하게 재구현했어요
   - AI와 협업할 때 한 번에 다 하려고 하지 말고 **작게 쪼개서 하나씩 검증**하는 게 중요합니다

2. **문제 상황을 구체적으로 설명하면 해결책이 나옴**
   - "DMG가 안 돼요" 대신 "개발 모드에서는 되는데 DMG로 설치하면 핫키가 안 먹혀요"라고 구체적으로 말하니 Claude가 4가지 원인을 찾아서 해결해줬어요

3. **AI에게 충분한 컨텍스트를 주면 정확도가 올라감**
   - 처음엔 AI가 프로젝트를 잘못 매칭했는데, 프로젝트 이름뿐 아니라 설명까지 컨텍스트로 주니까 정확도가 확 올라갔어요

4. **개발자 아니어도 Claude Code로 앱 만들 수 있다**
   - Electron이 뭔지도 몰랐는데 Claude가 전부 설명해주고 코드 짜줘서 macOS 앱까지 배포했어요
   - 제가 한 건 "이거 이렇게 해줘" 요청하는 것뿐입니다

### 이렇게 하면 안 돼요

1. **한 번에 너무 많은 기능을 추가하지 마세요**
   - Settings 기능을 한꺼번에 추가했다가 앱이 망가졌어요. 작게 쪼개서 단계별로 테스트하세요

2. **에러 메시지를 Claude에게 안 보여주지 마세요**
   - 콘솔 로그, 에러 메시지를 그대로 복사해서 보여주면 Claude가 정확한 원인을 찾을 수 있어요

3. **개발 모드에서만 테스트하지 마세요**
   - 개발 모드(`npm run start`)에서 잘 되도도 DMG로 빌드하면 안 될 수 있어요. 배포 환경까지 테스트하세요

## 🌍 다른 업무에 적용한다면?

이번 경험을 바탕으로 다른 자동화 아이디어들이 떠올랐어요:

- **채널톡/Slack 메시지 → Linear 이슈 자동 생성**: 특정 이모지 리액션을 달면 자동으로 이슈 생성
- **회의록 → Linear 액션 아이템 자동 추출**: 회의 내용에서 TODO를 찾아서 이슈로 생성
- **이메일 → Linear 이슈 변환**: 중요한 이메일을 포워딩하면 이슈로 저장
- **Linear 이슈 분석 → 업무 패턴 리포트**: 누적된 이슈를 분석해서 반복 업무 식별

핵심은 **"귀찮아서 미루던 기록을 자동화"**하는 거예요. Claude Code와 함께라면 충분히 가능합니다!

1. **Linear 이슈 자동 해결 에이전트**
   - 사람의 판단이 반드시 필요하지 않은 단순 업무를 AI에게 부분적으로 이관

   - 누적된 Linear 이슈를 분석해서 반복되는 업무 식별
   - 비효율적인 프로세스 자동 감지 및 개선 방향 제안

3. **다른 채널 자동 연동**
   - Slack, 채널톡, 이메일 등에서 발생하는 업무도 자동으로 Linear 이슈로 변환
   - 진짜 "단일 원천(Source of Truth)" 완성
